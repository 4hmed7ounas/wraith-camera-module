================================================================================

================================================================================

================================================================================
TROUBLESHOOTING PC TESTING
================================================================================

Problem: "No cameras detected"
Solution:
  1. Check if camera is connected
  2. Try: python -c "import cv2; cap=cv2.VideoCapture(0); print(cap.isOpened())"
  3. If False, your camera isn't recognized

Problem: "Module not found: ultralytics"
Solution:
  pip install ultralytics paddleocr paddlepaddle torch torchvision

Problem: "Slow performance on PC"
Expected - PC might be slower than optimized Pi due to:
  - No FP16 optimization on CPU
  - Different camera driver
  - System load

Note: Once deployed to Pi 5 with optimizations, will be much faster

Problem: "Import error: No module named rpi5_plate_reader"
Solution:
  Make sure rpi5_plate_reader.py is in same directory as test_on_pc.py

================================================================================
DIFFERENCES: PC vs RASPBERRY PI 5
================================================================================

Camera Capture:
   PC: Uses your built-in or USB webcam
   Pi: Uses CSI camera or USB webcam (faster)

Performance:
   PC: 5-10 FPS (typical, depends on CPU)
   Pi: 12-15 FPS (with optimizations)

Memory:
   PC: Usually 4GB+ available
   Pi: Needs swap configuration (2GB+)

Cooling:
   PC: Built-in cooling
   Pi: Might need heatsink + fan

Code Changes:
   NONE REQUIRED - Use same rpi5_plate_reader.py on both!
